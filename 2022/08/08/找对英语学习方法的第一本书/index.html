    为什么读这本书?—断断续续学习了很长一段时间，但感觉哪个地方出问题，但不清楚。故寻找相对正确有效的学习方法。

    这本书讲什么?—提出了部分学习英语的观念，并从实际情况举例证明这些观念错误无效，并探讨正确学习方法

    对这本书的评价是?—使对英语学习有一定的科学认识，认清了此前错误的行为及观念，实际效果需视最终实践结果。

    1.一些错误的观念或奇怪现象

    • 国外外语环境很快就会说好外语
    • 尽量多开口说
    • 英语不好因为单词量不够
    • 最好有人不断纠正错误
    • 语法熟练，句子就正确了
    • 多背诵，外语主要靠记忆力

    2.主要的一些理论及情景

    • 聋哑英语：首先应该是听力有问题，然后才是说不出来。（如英语电话号码听力记录例子，为何反映不过来无法记录？）
    • 听到的那些外国声音，是你无法理解的，是‘无效的输入’，即‘无效的声音’。因为不懂，所以没有任何学习效果，跟听到的噪音没太在区别。
    • 关于多开口说英语：当外语学习者在自己的外语交流能力很低时候，过早地，积极地在自然外语环境中跟native speaker交流，会扼杀语言学习过程中非常重要的‘沉默期’，反而造成语言水平难以提高（一到美国就主动和当地人交流，尽管自己英语能力还很低，但为了实现沟通的目的，努力地尝试用英语表达自己的意思，的确是很快实现了用简单的，自创的，东拼西凑出来的低水平中介语进行交流，英语能力被固化在一个低水平上。一旦这些‘自创语’实现交流目的，使用的人就会依赖于这些熟悉的自创语去交流，这时不断重复练习的还是他的‘自创语’。即使继续长期置身于外语环境中，也并不会再进步。这就是语言学中的‘石化现象’。
    • 没有学会英文思维，就不能跟着说话人的思路，地理解其连续表达的语言含义。不能实现听到英语时自然的英语思维，也不能做到随着交流的时行，自动‘确认’已经听到的词汇，和‘预测’即将听到的词汇，所以就无法实现正常速度的自然交流。
    • ‘听不清楚’，其实不是耳朵问题，是大脑认知问题。

    3.学习英语的四个阶段

    3.1 入门期 : 刚接触外语不适应，应注意减少大脑的负荷，集中精力只做很少的几件事。建议只学习简单常用的单词（名词为主，动词为辅），不学句子。同时应多听少说不读写。
    • 学习方法 :
      • 看图训音：使用罗塞达石碑APP学习，不要字幕，将‘说’和‘阅读’练习跳过。将图片与声音关联来。或可自己尝试使用软件应用将图片与声音结合。
      • T.P.R 全身肢体反应：把‘语言’和‘肢体动作’联系在一起，让大脑不同部位的共同协调训练建立语言思维，达到掌握外语的目的。
    • 入门阶段一些建议 :
      • 避免使用中文翻译，充分建立外语的反应
      • 不要试图说英语，只练听力
      • 开始时先只专注于学习单词，然后过渡到短语，不要着急学句子
      • 不要看英文的拼写，建议拼写的学习完全放弃
      • 不要怕听错，要放松和有信心
    3.2 提前期 :

    • 目标 :

      • (1) 听懂单句子，建立整句的英文思维
      • (2) 逐渐能听懂简单的、慢语速的整段英文
      • (3) 在抽调期后期按照一定的规范，开始尝试’输出‘简单英语
      • (4) 开始引入阅读材料，作为输入内容的扩充

    • 提高 : 本阶段不断提供有效的输入：视觉辅助输入，如模拟场景等

      • (1) 最理想的并不是动态的，而是静止画面
      • (2) 画面要求要简单，信息尽量少，像素要低
      • (3) 语言场景的故事情节应该是连续的，故事人物、背景、发展要有很强的相关性。连续有效提高’预测‘能力
      • (4) 根据情况，有的场景需要声音稍先于画面出现，靠画面带动理解声音，或相反
      • (5) 难度非常缓慢地逐渐加大。每个小段落完成后，需要有简单的总结、回放和小测试
      • (6) 如果有练习说的部分，要先听会才能说，对说的内容严格限定，一定是刚听过学过的内容，而且说的量最多只有听的十分之一。绝不能自由交谈
      • (7) 使用多种非标准音和标准发音对比的发音纠正体系
      • (8) 多媒体教学控制字幕。第一次不打开，中间重复听可出现字幕，最后又关掉

    • T.P.R 拓展教学 :

      • T.P.R with Objects(实物)，使用随意找到的东西，如苹果、香蕉等，使用这些物品关联动作或行为。如：This is an apple. Where is the apple?
      • T.P.R with Pictures ,使用照片或画面中的物品、人物、动作、情景等进行描述。亦可以扩展至更复杂背景信息进行描述。如穿着、年龄、职业、周围环境等
      • T.P.R with Stories，通过动作和道具表演一个简单的小故事，关键点是生动和有趣

      2.2.1 git bash 命令行下输入ssh-keygen -t rsa -C ‘你的邮箱地址’，然后回车，回车，再回车，一共3次回车键。
2.2.2 生成密钥后，在你电脑目录：C:\Users\你的计算机用户名.ssh 下，会生成两个文件：私钥：id_rsa 和公钥：id_rsa.pub。
2.2.3 输入cat id_rsa.pub 以查看并复制内容粘贴到https://github.com/settings/ssh ，点击添加新秘钥(New SSH key)，其中 Title 自己随便取，Key 就是把刚刚复制的都粘贴进来。
2.2.4 git config --global user.name "你的GitHub用户名"
2.2.5 git config --global user.email "你的GitHub注册邮箱

# Deployment
## 这里是重点，这里是修改发布地址，因为我们前面已经加了 SSH 密钥信息在 Github 设置里面了，所以只要我们电脑里面持有那两个密钥文件就可以无需密码地跟 Github 做同步。
## 需要注意的是这里的 repo 采用的是 ssh 的地址，而不是 https 的。分支我们默认采用 master 分支，以后你翅膀硬了要换其他也无所谓。
deploy:
 type: git
 repo:git@github.com:aidenhosea/aidenhosea.github.io.git
 branch: master
    • 2.4博客部署：
      hexo clean    #会清除缓存文件db.json及之前生成的静态文件夹public；
hexo g     #会重新生成静态文件夹public；
hexo s     #在本地服务器查看实际效果
hexo deploy    #因为之前已经安装了插件并且在博客配置文件中也配置好了，所以这个命令会在博客根目录下生成一个.deploy_git的文件夹，并 把本地生成的静态文件部署到qijian0503.github.io这个仓库中的master分支上；简写形式为hexo d；另外 hexo g 和 hexo d可以合并在一起写：hexo g -d
    • 2.5 域名配置：

      • 域名提供商系统设置（以阿里云为例）：登陆阿里云后台管理平台并设置以下：

        记录类型    主机记录    记录值
CNAME        @        aidenhosea.github.io
CNAME        www        aidenhosea.github.io

      • 在博客内 source 文件夹中添加 CNAME 文件，打开并在内填写阿里云购买的域名 aidenchan.top ，再保存。（将文件或图片放入 source 中才不会在部署时清理掉！！！）

      • 重新部署上传博客。
